## 📌 Option 3 — Technical (Best for Developer Audience)

> Just shipped: a **CrewAI multi-agent research pipeline** with a Streamlit UI.
>
> **What it does:**
> → Takes any topic as input
> → Agent 1 (Research Analyst) searches the web via Serper, gathers facts & sources
> → Agent 1 output is automatically passed to Agent 2 (Content Writer)
> → Agent 2 writes a structured Markdown blog post with citations
> → Final output downloadable directly from the UI
>
> **Stack:**
> - CrewAI (agent orchestration)
> - Groq API with LLaMA 3.3 70B (fast, free LLM)
> - Serper API (Google search for agents)
> - LiteLLM (LLM routing layer)
> - Streamlit (live visual dashboard)
>
> **Key concept I keep explaining to people:**
> CrewAI is NOT a wrapper around one LLM call. It's a **sequential multi-agent framework**. Each agent has a `role`, a `goal`, a `backstory` (for prompting), and assigned `tools`. The `Crew` runs tasks in order, passing outputs downstream automatically.
>
> No manual handoff code needed. That's the framework's job.
>
